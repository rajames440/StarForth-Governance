== Event Categories

=== Disposition Events

Recorded when documents enter the disposition workflow:

* **Routed to Pending:** Document classified and moved to Pending/[TYPE]/
* **Signer Notified:** Required signer(s) notified of pending document
* **Signature Collected:** Signature file received and verified
* **Escalation Triggered:** Day 1 reminder or Day 3 PM escalation triggered
* **Moved to Vault:** All signatures verified, document moved to [TYPE]/ vault

=== Signature Events

Recorded for all signature-related activities:

* **Signature Received:** New signature file detected
* **Signature Verified:** Signature cryptographically verified
* **Signature Invalid:** Signature failed verification (escalated to PM)
* **Signer Unauthorized:** Signature from unauthorized signer (escalated to PM)

=== Access & Control Events

Recorded for administrative changes:

* **Signer Added:** New authorized signer added to Signatures.adoc
* **Signer Revoked:** Signer authority revoked
* **Vault Locked:** Document marked immutable in vault
* **Document Archived:** Old version moved to SupportingMaterials/

=== Compliance Events

Recorded for regulatory and audit purposes:

* **Audit Trail Accessed:** SEC_LOG reviewed by auditor
* **Compliance Review:** Document compliance verification completed
* **Policy Updated:** Governance policy changed
* **Configuration Changed:** Pipeline configuration modified

== Security Properties

**Immutable:** This log should be treated as immutable once committed to master
**Protected:** Stored in Security/ directory, requires appropriate access controls
**Audit Ready:** Complete chain of custody for governance audits
**Timestamped:** All entries include UTC timestamp
**Attributed:** All entries credited to triggering system or user

== Viewing Audit Trail

Full audit trail is visible through git:

```bash
# See all disposition-related commits
git log --grep="disposition" --oneline

# See all security events
git log --grep="SEC" --oneline

# See complete history with times
git log --format="%h %ad %s" --date=iso
```

== Integration Points

This log is automatically updated by:

1. **Jenkinsfile.disposition** - Records routing, notification, and escalation events
2. **Signature Verification Script** - Records signature receipt and validation
3. **Vault Routing Script** - Records document movements to vault
4. **Administrator Commands** - Manual entries for policy/access changes
